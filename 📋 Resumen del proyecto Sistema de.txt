📋 Resumen del proyecto: Sistema de Evaluación de Personal
Arquitectura general
Backend: Python con Flask, API REST, almacenamiento en archivos CSV.

Frontend: React con Vite, estilos Bootstrap, rutas con React Router, peticiones con Axios.

Proxy: Configurado en vite.config.js para redirigir /api al backend en http://localhost:5000.

✅ Módulos completados
1. Configuración inicial
Backend: Flask, CORS, pandas, manejador CSV (utils/manejador_csv.py).

Frontend: Vite + React, Bootstrap, react-router-dom, axios, recharts.

2. Módulo de Candidatos
Backend: CRUD completo (routes/candidatos.py) con limpieza de NaN.

Frontend:

Listado (CandidatosList.jsx) con opciones de editar/eliminar.

Formulario (CandidatoForm.jsx) para crear/editar.

Servicio (services/candidatos.js).

3. Módulo de Cargos
Backend: CRUD (routes/cargos.py) con limpieza de NaN.

Frontend: Listado y formulario similares a candidatos.

4. Módulo de Evaluadores
Backend: CRUD con hash de contraseñas (routes/evaluadores.py).

Frontend: (pendiente de implementar en UI, pero el backend está listo).

5. Módulo de Tests
Backend: CRUD (routes/tests.py) con limpieza de NaN.

Frontend: Listado (TestsList.jsx) con botón "Ver preguntas" y formulario (TestForm.jsx).

6. Módulo de Preguntas
Backend: CRUD con validación de orden único por test (routes/preguntas.py), limpieza de NaN, soporte para opciones JSON.

Frontend:

Listado (PreguntasList.jsx) con selector de tests y enlace a creación.

Formulario (PreguntaForm.jsx) con campos dinámicos para opciones múltiples, validación de orden único, preselección de test desde URL.

Servicio (services/preguntas.js).

7. Módulo de Aplicación de Tests
Frontend:

Página de selección (AplicarTest.jsx) para elegir candidato y test.

Página de respuesta (ResponderTest.jsx) que muestra preguntas según tipo (Likert, múltiple, abierta) y envía respuestas.

Backend: Endpoint POST /api/respuestas para guardar respuestas (routes/respuestas.py).

8. Módulo de Resultados
Backend: Endpoint GET /api/resultados/<candidato_id>/<test_id> que calcula puntuación de la última respuesta (routes/resultados.py).

Frontend: Página (ResultadosTest.jsx) que muestra puntuación, porcentaje, detalle de respuestas y gráfico de barras con recharts.

🧪 Problemas resueltos
NaN en respuestas JSON: Se implementó limpiar_nan() en todos los endpoints para convertir NaN a null.

Validación de orden único: En preguntas, se evita duplicados por test.

Preselección de test: Al crear pregunta desde la lista de un test, el campo test_id se autoselecciona.

Ancho de tabla: Se usó tableLayout: fixed y estilos para que el texto largo se ajuste.
